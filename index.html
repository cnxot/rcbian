Project Brief: Development of a Mobile-First, Interactive Promotional Landing Page Application

1. Objective

This project brief outlines the development of a mobile-first, interactive promotional landing page themed around the Royal Challengers Bengaluru (RCB) cricket team. The primary objective is educational: to provide a hands-on learning experience in front-end development, user interface design, and JavaScript interactivity. The application will simulate common "deceptive patterns" found in online promotions, not to endorse them, but to deconstruct their mechanics for learning purposes. The final stage of the user journey will explicitly reveal the simulation and provide educational context on user psychology and ethical web design.

Ethical Mandate: This project is strictly for educational purposes. It aims to teach developers how to recognize and understand potentially manipulative design patterns to build more transparent and ethical web applications. It does not endorse the use of deceptive tactics in live projects.

3. Technical Specifications

  • Frameworks: Vanilla HTML5, CSS3, and JavaScript (ES6+). No external libraries or frameworks (like React or Vue) are required to keep the focus on core web technologies.
  • Architecture: A single-page application (SPA) architecture, where content for different steps is dynamically shown/hidden using JavaScript. This avoids page reloads and creates a seamless user experience.
  • Responsiveness: A mobile-first approach is mandatory. All components must be designed for small screens and scale gracefully to desktop sizes using media queries.

4. User Journey & Component Breakdown

The user will progress through a series of steps designed to simulate a promotional contest.

  • Visuals: Full-screen splash with an RCB-themed background. A large, animated "PLAY BOLD, WIN BIG!" headline.
  • Text: "Exclusive Chance! Win 2 VIP Tickets to the next RCB match at Chinnaswamy Stadium!"
  • Functionality: A single, prominent button with the text "TAP TO PLAY!". Clicking this button triggers the next step.

Step 2: The Illusion of Progress

  • Visuals: The button from Step 1 is replaced by a progress bar.
  • Text: "Verifying your entry... Analyzing your fan level..."
  • Functionality: The progress bar animates from 0% to 100% over 3-4 seconds to build anticipation. This uses a simple `setInterval` function.
// Sample code for the progress bar
function startProgressBar() {
  let width = 1;
  const progressBar = document.getElementById('myBar');
  const interval = setInterval(() => {
    if (width >= 100) {
      clearInterval(interval);
      showNextStep(); // Transition to the next part
    } else {
      width++;
      progressBar.style.width = width + '%';
      progressBar.innerHTML = width + '%';
    }
  }, 30); // 30ms interval for smooth animation
}
